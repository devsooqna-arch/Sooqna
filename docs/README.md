# Sooqna Documentation

This folder is the current source of project documentation. It replaces old phase notes, generated PDFs, screenshots, and temporary planning files.

## Reading Order

1. `architecture.md` - system boundaries and data ownership.
2. `local-development.md` - local setup and daily commands.
3. `api-reference.md` - backend modules and route map.
4. `admin-dashboard.md` - admin features, roles, and moderation.
5. `deployment-operations.md` - CI/CD, deployment, database, and runtime operations.
6. `security.md` - auth, authorization, rate limits, uploads, and secrets.
7. `product-roadmap.md` - recommended product improvements.

## Documentation Rules

- Keep docs current with production code.
- Do not commit generated reports, screenshots, logs, or exports.
- Do not include secrets, private keys, database URLs, or Firebase API credentials.
- Prefer one maintained document over many stale phase documents.
